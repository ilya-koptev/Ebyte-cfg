# Работа с дашбордом

Инструмент создаёт виртуальное устройство **«EBYTE configurator»** в homeui
(вкладка **Устройства**). Все контролы — на английском, как в фирменном приложении.

Порядок (сверху вниз): **Search** · **Status** · **Default** · сеть · serial ·
Modbus · Link 1 · Link 2 · Net AT · **Write**.

## 1. Подключение устройства

- Воткни Ethernet устройства в **Ethernet 2** контроллера (конфиг-порт).
- Подключи его **RS-485 (A/B)** к **RS485-2** контроллера.
- Запитай EBYTE от **V_OUT** — это нужно, чтобы инструмент мог перезагрузить его
  для применения настроек и прогнать тест RS-485.

После подключения дай пару секунд на загрузку.

## 2. Search (поиск)

Нажми **Search**. Форма очистится, просканируется Ethernet 2 и заполнятся все поля.
Статус покажет, например, `found: NE2-D11 192.168.69.58`. Если ничего не найдено —
`device not found` (проверь кабель / что устройство догрузилось).

## 3. Default (шаблон, опционально)

Нажми **Default**, чтобы загрузить в форму заведомо рабочий шаблон (IP
`192.168.69.10`, 9600 8N2, прозрачный Modbus, Link 1 = TCP server, DHCP off, Net
AT off …). Статус: `template loaded — review and press Write`. В устройство пока
ничего не пишется.

## 4. Правка нужного — например, IP

Измени любое поле, напр. **Local IP** → `192.168.69.20`. Поля-перечисления (Work
Mode, Baud, Parity, DHCP, MODBUS Getway …) — **выпадающие списки**. Как только
что-то меняешь, статус показывает `template changed`.

Видимость полей зависит от режима линка: **TCP server** прячет remote-поля
(показывает `—`), **Disable** прячет все поля этого линка. Прочерки `—` в
устройство не пишутся.

## 5. Write (запись)

Нажми **Write**. Инструмент:

1. запишет все поля во **флеш** устройства,
2. **перезагрузит** устройство (передёрнет V_OUT, ~15 с),
3. **прочитает обратно и сверит** значения,
4. прогонит **тест моста RS-485** (serial ↔ TCP).

## 6. Убедиться, что всё ок

Смотри на **Status**. Хороший результат:

```
done. config matches. RS-485: OK — TCP->485:True  485->TCP:True @9600 8N2
```

- `config matches` — все записанные значения прочитались обратно верно.
- `RS-485: OK` — данные прошли в обе стороны по RS485-2 на настроенной скорости/формате.
- `RS-485: skipped (...)` — если Link 1 не в режиме **TCP server** (тесту нужен
  слушающий локальный порт) или режим не прозрачный.
- `mismatch: <поля>` — эти поля не применились (проверь и запиши ещё раз).

## Заметки

- Изменения serial / Modbus / Link вступают в силу только **после перезагрузки** на
  шаге 5 — поэтому Write всегда передёргивает питание.
- Правка **databit** безопасна (только 5–8); недопустимые значения инструмент
  отклоняет.
- Вернуть устройство к чистому состоянию: **Default → Write**.
